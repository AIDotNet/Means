Means separates object bytes from object visibility. Data files may exist before commit, but they become visible through the S3 API only after metadata commit succeeds.

## Metadata Responsibility

`MeansLogDb` manages namespaces for buckets, objects, versions, multipart uploads, parts, AccessKeys, policies, settings, audit records, tasks, and EC profiles.

## Strong Read/Write Semantics

HEAD, GET, and LIST read only committed current object pointers. Overwrites replace the current pointer in a single batch so readers never observe a half-committed object.
