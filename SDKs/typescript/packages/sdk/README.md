# @means/sdk

Browser-safe shared SDK for Means object storage.

This package has no credential or signing implementation. Use it for anonymous bucket/object requests and for executing presigned URLs generated by a trusted server or by `@means/sdk-node`.

## Install

```bash
npm install @means/sdk
```

## Anonymous client

```ts
import { MeansClient } from "@means/sdk";

const client = new MeansClient({
  endpoint: "https://api.means.local",
  addressingStyle: "path"
});

const object = await client.getObject({
  bucket: "public-assets",
  key: "logo.svg"
});

const svg = await object.response.text();
```

## Presigned URL execution

```ts
import { getObjectFromUrl, putObjectToUrl } from "@means/sdk";

const download = await getObjectFromUrl(presignedGetUrl);
const bytes = await download.response.arrayBuffer();

await putObjectToUrl(presignedPutUrl, file, {
  contentType: file.type
});
```

## Multipart upload

`MeansClient` exposes low-level multipart methods and a browser-safe high-level helper for `Blob`, `File`, `ArrayBuffer`, and `Uint8Array` bodies:

```ts
await client.uploadObjectMultipart({
  bucket: "uploads",
  key: file.name,
  body: file,
  contentType: file.type
});
```

The helper uses 16 MiB parts by default and aborts the multipart upload if any part fails. Browser code still needs anonymous policy access or presigned URLs; this package does not contain secret-key signing.
