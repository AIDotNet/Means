# @means/sdk

Browser-safe shared SDK for Means object storage.

This package has no credential or signing implementation. Use it for anonymous bucket/object requests and for executing presigned URLs generated by a trusted server or by `@means/sdk-node`.

See `../../../examples/typescript/browser-presigned-upload.ts` for a browser upload example that consumes presigned URLs from an application backend.

## Install

```bash
npm install @means/sdk
```

## Anonymous client

```ts
import { MeansClient } from "@means/sdk";

const client = new MeansClient({
  endpoint: "https://api.means.local",
  addressingStyle: "path"
});

const object = await client.getObject({
  bucket: "public-assets",
  key: "logo.svg"
});

const svg = await object.response.text();
```

## Presigned URL execution

```ts
import { getObjectFromUrl, putObjectToUrl } from "@means/sdk";

const download = await getObjectFromUrl(presignedGetUrl);
const bytes = await download.response.arrayBuffer();

await putObjectToUrl(presignedPutUrl, file, {
  contentType: file.type
});
```

## Multipart upload

`MeansClient` exposes low-level multipart methods and a browser-safe high-level helper for `Blob`, `File`, `ArrayBuffer`, and `Uint8Array` bodies:

```ts
await client.uploadObjectMultipart({
  bucket: "uploads",
  key: file.name,
  body: file,
  contentType: file.type
});
```

The helper uses 16 MiB parts by default and aborts the multipart upload if any part fails. Browser code still needs anonymous policy access or presigned URLs; this package does not contain secret-key signing.

Low-level multipart calls also include `uploadPartCopy`, `listParts` pagination, and bucket-level `listMultipartUploads` pagination/common-prefix fields.

## Versioning and lifecycle

```ts
await client.setBucketVersioning({ bucket: "uploads", status: "Enabled" });

const versions = await client.listObjectVersions({
  bucket: "uploads",
  prefix: "raw/"
});

const oldObject = await client.getObject({
  bucket: "uploads",
  key: "raw/photo.jpg",
  versionId: versions.versions[0].versionId
});

await client.putBucketLifecycle({
  bucket: "uploads",
  configuration: {
    rules: [
      {
        id: "expire-temp",
        status: "Enabled",
        prefix: "tmp/",
        expirationDays: 7,
        abortIncompleteMultipartUploadDays: 1
      }
    ]
  }
});
```

Lifecycle day values must be positive integers. Use `deleteBucketLifecycle` to remove all rules.

## Tagging, CORS, and notification

```ts
await client.putObjectTagging({
  bucket: "uploads",
  key: "raw/photo.jpg",
  tags: { kind: "raw" }
});

const tagging = await client.getObjectTagging({
  bucket: "uploads",
  key: "raw/photo.jpg"
});

await client.putBucketCors({
  bucket: "uploads",
  xml: "<CORSConfiguration><CORSRule><AllowedOrigin>*</AllowedOrigin><AllowedMethod>GET</AllowedMethod></CORSRule></CORSConfiguration>"
});
```

Bucket CORS and notification APIs accept raw XML. The SDK validates the root element and leaves detailed S3 XML compatibility to the service.
